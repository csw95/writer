# Canon Facts Methodology

幕后事实库用于记录小说世界中“真实发生了什么”。它服务作者和系统，不等同于正文前的读者说明。

## 核心定义

- **事实库（canon/facts.md）**: 记录历史真相、暗线真相、身份秘密、能力规则、阵营事实、人物过去、物品真相和制度底层规则。
- **伏笔台账（open-loops/loops.md）**: 记录读者侧可感知的悬念、线索、误导和回收计划。
- **世界观（world/worldbuilding.md）**: 记录世界中公开可用的规则边界、力量体系和社会结构。

三者关系：

- 世界观回答“这个世界允许什么”。
- 事实库回答“真实发生过什么、谁知道什么、真相何时显露”。
- 伏笔台账回答“读者看见了什么问题，以及何时回收”。

## 必须登记的事实

凡符合任一条件，必须先进入 `canon/facts.md`，再进入章节规划：

- 会影响主角选择、阵营站位、证据链、牺牲、背叛、救赎或恋爱推进。
- 会解释关键历史事件、家族旧案、战争结果、灾难源头、制度成因或势力仇怨。
- 会改变读者对人物身份、动机、能力来源、过去行为或当前立场的理解。
- 会决定能力规则、代价、禁忌、上限、资源来源或特殊物品真相。
- 会承担长期伏笔、误导、反转、回收或终局因果功能。

## 事实登记粒度

事实必须写到可验证，不允许只写氛围词。

合格示例：

- `FACT-001`: 十七年前的城火不是天灾，而是巡夜司为封锁异化源头主动焚城；公开版本是山火失控。
- `FACT-002`: 主角体内的石纹不是天赋，而是旧阵眼碎片，使用越多越接近被阵眼同化。

不合格示例：

- “主角身世很神秘”
- “旧案背后另有隐情”
- “这个能力有代价”

## 知情状态

每个事实必须标明：

- 谁完整知情。
- 谁只知道部分。
- 谁被公开版本或假证据误导。
- 谁完全不知情。
- 本章是否改变某个角色的知情状态。

角色的知情状态会直接约束正文对话和选择。角色不能说出自己不知道的真相，也不能基于未获知事实做出精准判断。

## 显露层级

章节计划必须明确本章对事实的显露层级：

- **不显露**: 事实只作为幕后因果，不进入正文。
- **暗示**: 正文出现异常、细节、反常行为或碎片线索，但不解释真相。
- **误导**: 正文强化公开版本或错误解释，必须与事实库中的公开版本一致。
- **部分揭露**: 读者或角色获得真相的一部分，但不能越过计划揭露窗口。
- **公开揭露**: 真相被明确说出或证实，必须进入 `open-loops` 回收或状态更新。
- **反转修正**: 公开版本被推翻，必须有已登记事实和伏笔支撑。

## 改写规则

事实库允许增量补充，但禁止临场推翻核心事实。

可以做：

- 补充新事实的影响范围、知情人、关联伏笔。
- 更新“当前显露状态”。
- 在未违背既有事实的情况下增加更细的历史过程。

不可以做：

- 为了解决当前章节困难，临时改写已登记真相。
- 把未登记的隐藏事实直接写入正文作为反转。
- 让角色突然知道未显露事实。
- 把公开版本当作真实事实覆盖原真相。

确需修订核心事实时，必须先停止正文生成，回到内容补齐，并在事实条目中记录修订原因、影响范围和需要回查的章节。

## 与伏笔的绑定

不是所有事实都需要伏笔，但所有长期伏笔都必须能绑定至少一个事实或明确的未来答案。

- `LOOP-*` 负责追踪读者看到的问题。
- `FACT-*` 负责保存真实答案。
- chapter_plan 引用暗线、历史旧案、身份秘密或能力代价时，必须同时标明关联的 `FACT-*` 和 `LOOP-*`（如有）。

## 开篇规则

第一章不得把事实库当作百科说明使用。

允许：

- 通过冲突、压迫、异常、代价、错判和公开版本显露事实的一角。
- 让读者感到“这里有问题”，但不解释全部历史。

禁止：

- 开篇直接讲述完整历史真相。
- 用作者旁白暴露主角身世、幕后黑手或终局因果。
- 为解释事实库而打断当前场景。
