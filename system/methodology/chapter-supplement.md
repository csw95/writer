# Chapter Supplement Methodology

章节后补充说明规范。用于帮助审阅人、接力作者和后续自动化理解本章出现的专有名词、规则口径、证据链和复杂剧情逻辑。

## 定位

- 补充说明属于章节后信息，不属于正文，不计入正文字数。
- 补充说明服务审阅和接力，不替代正文中的必要因果、情绪、冲突和信息显露。
- 正文读者必须能从正文读懂本章核心事件；如果核心剧情只能靠补充说明才成立，应回到正文小修或重写。
- 补充说明不得新增 `canon/facts.md` 未登记的核心事实，不得新增 `canon/items.md` 未登记的关键物品功能，不得越过本章 `canon_fact_plan`、`key_item_plan`、`facts-active.md`、`items-active.md` 和当前 Volume / Arc 的显露/操作上限。

## 输出位置

每章正文文件末尾，在 `## 已显露事实` 之后追加：

```markdown
## 名词与剧情说明

### 名词说明
- **{名词}**: {本章已显露层级内的简明解释；可标注关联 FACT/ITEM/LOOP}

### 剧情详解
- **{问题或场景}**: {解释本章中的因果、规则碰撞、角色判断、证据链或后续影响}

### 审阅边界
- **暂不展开**: {说明本章不应提前解释的内容，避免审阅时误判为漏洞}
```

如果本章没有复杂专有名词，也必须保留 `## 名词与剧情说明`，可写“本章无新增复杂名词；沿用前文说明”，并补一条本章剧情承接说明。

## 内容要求

每章通常写 3-8 条，优先覆盖：

1. 本章首次出现或容易误读的专有名词、制度名、术法名、关键物品名、地点名。
2. 本章关键规则碰撞，例如契文口径冲突、能力代价、身份记录异常。
3. 本章证据链、关键物品状态与伏笔关系，例如某物证、证人、异常动作如何服务后续章节。
4. 本章角色判断依据，尤其是主角为什么能看出破绽、为什么作出某个选择。
5. 审阅边界：哪些内容是刻意暂不解释，后续在哪类窗口回收，但不提前剧透终局真相。

每条说明应短而准：

- 名词说明控制在 1-3 句。
- 剧情详解控制在 2-4 句。
- 能标注 `FACT-*` / `LOOP-*` 时标注，但不得粘贴事实库完整幕后真相。

## 禁止项

- 禁止在补充说明里提前公开未显露真相、终局反转、人物隐藏身份全貌或后续章节答案。
- 禁止用补充说明新增正文中不存在的事件、人物动机、物品能力、关键物品状态变化或世界规则。
- 禁止用补充说明弥补正文缺失的关键动作、核心选择、证据来源或因果后果。
- 禁止把补充说明写成百科式世界观长文。
- 禁止把补充说明写成审稿意见、修改建议或评分报告。

## 审阅标准

Reviewer 必须检查：

- 是否存在 `## 名词与剧情说明`。
- 名词解释是否与 world、canon、items、chapter_plan 和本章正文一致。
- 剧情详解是否只解释已写出的内容，没有替正文补剧情。
- 审阅边界是否防止提前剧透，而不是掩盖真实逻辑漏洞。
- 补充说明是否没有计入实际正文字数。
