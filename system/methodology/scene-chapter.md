# Scene And Chapter Methodology

场景与章节规范。用于 Planner、Writer 和 Reviewer。

## 场景五要素

每个关键场景必须包含：

1. **目标**: 角色想做什么。
2. **阻碍**: 谁或什么阻止他。
3. **危机选择**: 他必须在什么代价之间选择。
4. **高潮行动**: 他做出什么不可撤销的行动。
5. **结果变化**: 局势从什么值变成什么值。

## 场景呈现要素

每个关键场景还必须遵守 `system/methodology/prose-style.md`，并在 chapter_plan 中声明：

1. **呈现焦点**: 读者最该看见的画面、人物动作或物件状态。
2. **感官锚点**: 至少 2 个与冲突相关的声音、光线、气味、温度、触感、疼痛或空间距离线索。
3. **动作节拍**: 关键动作按触发、反应、碰撞/受阻、代价和余波拆解。
4. **句式节奏**: 根据场景压力选择短促、压迫、舒缓、克制、爆发或冷硬等策略。
5. **禁止概括**: 不得用一句话跳过的关键动作、情绪或局势变化。

## 章节基础模板

```text
角色想做一件事
-> 遇到阻碍
-> 采取行动
-> 局势变化
-> 得到结果/付出代价
-> 引出下一章问题
```

## 情绪曲线

每章必须声明一种主要情绪曲线：

- 压抑 -> 释放
- 误解 -> 证明
- 低估 -> 反转
- 失去 -> 夺回
- 危机 -> 突破
- 期待 -> 兑现
- 谜题 -> 揭晓
- 羁绊 -> 守护

## 章末钩子类型

| 类型 | 用法 |
|------|------|
| 危机钩子 | 新危险立即出现 |
| 信息钩子 | 关键名字、物品、真相露出 |
| 反转钩子 | 已知局势被推翻 |
| 选择钩子 | 主角必须做代价选择 |
| 爽点钩子 | 主角将要高光反击 |
| 误会钩子 | 外界误判主角，引出冲突 |

## 单章验证

一章必须满足：

- 正文字数符合 chapter_plan.length_plan。
- 每个关键场景承接 length_plan.scene_budgets 的字数预算。
- 至少 1 个冲突点。
- 至少 1 个状态变化点。
- 至少 1 个剧情变量推进。
- 至少 1 个类型承诺兑现。
- 章末钩子不能与上一章重复。
- 不能只靠信息解释推进。
