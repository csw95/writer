# Novel Categories

小说分类规范。初始化、章节规划和审阅时必须引用。

分类用于确定平台货架、目标读者和基础期待；类型承诺用于确定章节级快感兑现。二者必须一致，但不能互相替代。

## 分类字段

每本小说在 `premise` 中必须声明：

```yaml
classification:
  channel: 男频/女频/通用
  category: 主大类
  subcategory: 细分类
  secondary_categories: [可选交叉类，最多2个]
  tags: [2-6个卖点标签]
  audience_expectation: 核心读者持续追读期待
  forbidden_drift: 禁止偏离的分类边界
```

## 分类原则

1. **频道不是作者或主角性别**: 男频/女频只表示首要读者期待和平台分发口径。
2. **主分类只能有一个**: 章节、卷和 Arc 都必须优先服务主分类。
3. **细分类必须可执行**: 不得只写“都市”“言情”“幻想”，必须写到可指导剧情机制的层级。
4. **标签补充卖点**: 标签用于声明金手指、关系模式、爽点机制、世界元素或叙事结构。
5. **交叉分类有主次**: 例如“仙侠言情”若主快感是情感拉扯和关系递进，归女频幻想言情；若主快感是修炼升级和战斗压制，归男频仙侠。
6. **禁止分类漂移**: 连续 3 章偏离主分类核心期待，Reviewer 必须标记“分类漂移风险”。

## 通用分类

以下分类可进入男频、女频或通用频道，必须根据主快感决定最终频道：

| 大类 | 频道判断 |
|------|----------|
| 悬疑 | 以谜题、案件、恐惧为主可通用；以战斗升级为主偏男频；以关系信任和女性成长为主偏女频 |
| 科幻 | 以概念推演、技术战争为主偏男频或通用；以星际关系、成长和情感选择为主偏女频 |
| 历史 | 以争霸战争、制度升级为主偏男频；以宅斗宫斗、权谋情感为主偏女频 |
| 现实 | 以行业奋斗、商业逆袭为主偏男频；以女性职场、家庭关系、自我成长为主偏女频 |
| 轻小说 | 以战斗冒险、系统规则为主偏男频；以关系互动、治愈陪伴为主偏女频；以日常群像可通用 |

## 分类示例

### 男频仙侠

```yaml
classification:
  channel: 男频
  category: 仙侠
  subcategory: 凡人流
  secondary_categories: [玄幻]
  tags: [宗门, 修炼升级, 资源争夺, 低阶逆袭]
  audience_expectation: 看主角在严苛修炼秩序中以代价换突破，逐步撬动宗门和大势力格局。
  forbidden_drift: 不得连续多章脱离修炼、资源、斗法和势力压力。
```

### 女频古言

```yaml
classification:
  channel: 女频
  category: 古代言情
  subcategory: 女强权谋
  secondary_categories: [穿越重生]
  tags: [重生复仇, 朝堂, 强强, 掉马]
  audience_expectation: 看女主利用前世信息差反击旧敌，在权谋和情感选择中夺回主动权。
  forbidden_drift: 不得把主线长期写成单纯升级打怪，情感关系和权力博弈必须持续推进。
```

## 审阅规则

Reviewer 必须检查：

- 频道、大类、细分类是否在 `premise` 中明确。
- 本章快感是否服务主分类，而不是只服务交叉分类。
- 标签是否在剧情中有可感知信号。
- 是否出现连续 3 章分类漂移。
- 新增设定是否改变原分类定位；如改变，必须退回初始化或结构层重新确认。
