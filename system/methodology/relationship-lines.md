# Relationship Lines Methodology

关系线主次编排规范。适用于初始化、长期结构、Volume / Arc 规划、章节规划、正文生成、审阅和状态更新。

## 目标

关系线不是临场调味，而是长篇结构轴。情感线、手足线、同袍线、师徒线、敌友线和阵营信任线，都必须提前确定主次、功能、阶段节点和越界边界。

关系线必须解决三个问题：

1. 这条关系为什么存在，服务主线、人物弧光、类型承诺还是情绪回报。
2. 这条关系在 Part / Volume / Arc / Chapter 哪些节点发生变化。
3. 这条关系不能做什么，避免抢主线、分类漂移或临场强行推进。

## 关系线分级

每条长期关系线必须标记一个等级：

- **主关系线**: 与全书主题和主线目标绑定，通常贯穿全书。例如主角与制度、主角与核心共同体。
- **一级副线**: 多卷反复推进，能改变主角选择、同盟结构或长期命运。
- **二级副线**: 服务某个 Part / Volume 的情绪、证据、救援、背叛、牺牲或信任建立。
- **章节点缀线**: 只在单章增强情绪，不改变人物关系和命运方向。
- **禁用线**: 与分类定位冲突、会抢主线或现阶段没有结构承载的关系推进。

若一条点缀线连续 3 次出现，或开始影响主角选择、资源分配、阵营站位、牺牲、恋爱、背叛、离队、救赎，就必须升级为二级副线并写入结构规划。

## 关系类型

常用类型：

- **情感 / 恋爱线**: 暧昧、信任、守护、误会、选择、确认关系。
- **手足 / 亲情线**: 家人失散、救援、愧疚、保护、和解、传承。
- **同袍 / 盟友线**: 互救、并肩、分歧、重盟、共同承担代价。
- **师徒 / 引路线**: 试探、传授、代价警告、传承、牺牲。
- **敌友 / 镜像线**: 追捕、利用、合作、道路分歧、有限让步。
- **阵营 / 共同体线**: 团队信任、制度设计、内部冲突、组织成型。

每条关系线只能有一个主功能，其他功能作为补充，不能把同一条线同时写成恋爱、证据、师徒、权谋和救赎的全能工具。

## 必填字段

长期关系线必须在 `characters/cast.md` 或结构文件中具备：

- **关系线ID**: `REL-类型-编号`，例如 `REL-ROM-001`、`REL-COMRADE-001`、`REL-MAIN-001`、`REL-MIRROR-001`。
- **关系类型**: 情感 / 手足 / 同袍 / 师徒 / 敌友 / 阵营等。
- **优先级**: 主关系线 / 一级副线 / 二级副线 / 点缀线。
- **关联角色**: 至少 2 个角色或 1 个角色 + 1 个阵营 / 制度。
- **叙事功能**: 服务主线、人物弧光、证据链、情绪回报、势力成长、主题论证中的哪一项。
- **当前阶段状态**: 陌生、试探、互利、信任、裂痕、重盟、确认、决裂等。
- **阶段节点**: Part / Volume / Arc / Chapter 的预计推进点。
- **触发条件**: 关系变化必须由什么事件、选择、代价或证据触发。
- **越界禁止**: 不能突然恋爱、不能无动机背叛、不能为情绪牺牲主线逻辑等。
- **最晚确认章节**: 若关系性质存在分支，必须在此章前确认。

## 结构层级要求

### 全书 / Part

必须回答：

- 哪些关系线贯穿全书。
- 哪些关系线只服务某个 Part。
- 哪些关系线会改变主角的终局选择。
- 哪些关系线不得越过主类型承诺。

### Volume

每卷必须列出本卷推进的关系线：

- 本卷关系起点。
- 本卷关系终点。
- 本卷关键节点。
- 本卷不推进但需要保持连续性的关系线。

### Arc

每个 Arc 必须列出章节级关系节拍：

- 哪些章节推进关系。
- 哪些章节只维持关系。
- 哪些章节禁止推进关系，以免抢夺主冲突。
- 关系变化是否对应证据、资源、救援、选择或代价。

### Chapter

每个 chapter_plan 必须声明本章关系线操作：

- **推进的关系线**: 无 / 关系线ID。
- **关系状态变化**: 无变化 / 从 A 到 B。
- **触发事件**: 本章哪个事件造成变化。
- **主次约束**: 该关系推进如何服务本章主目标和类型承诺。
- **禁止越界检查**: 是否存在未规划恋爱、背叛、牺牲、决裂、确认关系等重大推进。

## 主次约束

关系线必须服从 `premise` 的 channel / category / subcategory。

- 男频玄幻、仙侠、都市升级等：情感线通常不得成为第一主驱动，除非 premise 已明确为情感向；更适合写成信任、并肩、守护、代价和慢热确认。
- 女频言情、幻想言情、古言、现言等：情感关系可以是主驱动，但仍必须提前编排阻碍、阶段选择和关系代价。
- 多元情感、群像陪伴等：关系线可以作为主要情绪回报，但必须有阶段目标和冲突，不得写成无推进日常。

## 门禁规则

以下情况必须回到结构补齐，禁止直接写正文：

- 长期关系线未登记，却在章节中推动恋爱、背叛、牺牲、决裂、离队、救赎或阵营站位。
- 男女主 / 双主角 / 核心同袍关系未在 Volume / Arc 中安排节点，却在正文中突然升温或确认。
- 手足、亲情、同门、同袍关系承担证据、救援、牺牲或人物转折功能，却没有在人物和 Arc 门禁中登记。
- 关系线连续抢夺主分类快感，导致升级、解谜、权谋、战斗、经营等主承诺失焦。
- 审阅发现关系变化无法由事件、选择、代价或证据触发。

## 状态更新

每章完成后，State Manager 必须更新：

- 本章推进的关系线ID。
- 关系状态变化。
- 触发事件。
- 关系强度或信任度变化。
- 下一次计划推进节点。
- 是否产生新的关系伏笔。

关系线如果连续 10 章没有推进，应在下一次 Volume / Arc 规划时确认：继续保持、降级、暂停或回收。
