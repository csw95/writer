# Review Rubric

长篇连载质量门禁。用于 Reviewer AI、单章自动化运行前检查、阶段审稿和人工抽查。

## 单章评分维度

评分维度、权重、综合分公式和通过阈值的唯一权威来源是 `system/schemas/scoring.schema.md`。本文件只同步展示当前版本；若两处不一致，以 `scoring.schema.md` 为准，并必须修正本文件。

| 维度 | 权重 | 说明 |
|------|------|------|
| conflict_strength | 15% | 冲突是否明确、有压迫、有升级 |
| pacing_speed | 12% | 是否紧凑，是否存在无推进段落 |
| satisfaction_level | 12% | 是否产生情绪回报 |
| hook_strength | 8% | 章末是否驱动下一章 |
| genre_promise | 8% | 是否兑现类型承诺，并服务既定分类 |
| logic_consistency | 10% | 因果、时间线、战力、事实库和角色知情状态是否自洽 |
| world_fidelity | 5% | 是否遵守世界规则、能力/制度事实和禁止改写项 |
| character_consistency | 6% | 角色画像、行为、命运推进和关系线操作是否符合设定 |
| setup_payoff | 5% | 伏笔是否有推进或合理埋设，长期伏笔是否绑定 Fact ID 或明确未来答案 |
| causal_chain | 7% | 场景与关键剧情因果闭环质量 |
| continuity_control | 6% | 时间线、地点、人物位置和章节承接稳定性 |
| power_resource_consistency | 4% | 战力、能力、资源、道具、伤势和地位守恒 |
| novelty | 2% | 是否避免重复爽点和桥段 |

## 叙事表现门禁

叙事表现独立评分，暂不进入综合分权重，但会决定章节是否需要小修或重写。

| 维度 | 说明 |
|------|------|
| scene_texture | 场景空间、压力、物件状态和氛围是否可感知 |
| action_clarity | 关键动作是否清晰、连续、有碰撞、有代价 |
| prose_vividness | 语言是否具体、有辨识度，避免抽象总结 |
| pov_embodiment | 是否贴合 POV 的注意力、身体感受、知识边界和情绪状态 |

表现门禁判定：

- 任一项 < 5：对应场景必须重写。
- 任一项为 5-6：正文必须小修后才能进入 State Manager。
- 全部 >= 7：叙事表现通过。
- 表现修正不得改变事件结果、关系变化、事实显露、战力资源、时间地点和后续影响。

## 开篇额外评分

第一章额外检查：

- opening_conflict: 前 500 字是否建立冲突或异常。
- protagonist_clarity: 主角身份、处境、目标是否清楚。
- premise_signal: 一句话卖点和分类定位是否在第一章有可感知信号。
- worldbuilding_reveal: 世界观是否通过冲突、压迫、选择、代价或异常逐步显露，且没有百科式开场。
- chapter_2_pull: 章末是否自然拉动第二章。

任一开篇额外项低于 7，第一章不得作为自动化续写起点。

## 判定规则

- 综合分 >= 7.5 且核心维度 >= 6: 通过。
- 综合分 7.0-7.4: 需小修后通过。
- 综合分 < 7.0: 需重写。
- 任何 P0 问题: 阻塞，回到规划或初始化补齐。
- 叙事表现门禁任一项 < 5: 对应场景需重写。
- 叙事表现门禁任一项为 5-6: 小修后通过。
- 正文字数不符合 `length_plan`: 视偏差程度判定为小修、需重写或拆章。
- 连续性或伏笔失控: 阻塞，不允许进入自动化续写。
- 分类定位连续 3 章漂移: 阻塞，回到规划或初始化确认。
- 未建档关键人物承担转折、伏笔或长期剧情功能: 阻塞，回到人物画像补齐。
- 背叛、死亡、洗白、恋爱推进、离队、升阶等重大人物转折未在 Volume / Arc 人物命运中预设: 阻塞，回到规划。
- 情感、手足、同袍、师徒、敌友或阵营信任等关系线未在关系线台账和当前 Volume / Arc 中预设，却承担恋爱升温、和解、决裂、背叛、牺牲、离队、救赎或阵营转向: 阻塞，回到关系线编排。
- 未登记核心事实承担历史因果、身份秘密、能力真相、暗线反转或伏笔回收功能: 阻塞，回到事实库补齐。
- 正文违背 `canon/facts.md`、让角色知道未显露事实、或公开超过当前 Volume / Arc 揭露窗口的信息: 阻塞，回到事实库、事实显露计划或章节规划。

## 单章自动化运行前检查

每次自动化任务只允许生成一个章节或处理一个 `pending_action`。运行前必须确认：

- 当前 state 与最新章节一致。
- state 的运行控制字段完整，`run_lock` 可用。
- `pending_action` 已按路由处理；不为 none 时不得生成章节。
- `last_run_status` 不是未恢复的 partial 状态。
- `next_chapter_to_generate = last_completed_chapter + 1`。
- canon/facts.md 的显露状态、读者已知和角色知情变化与最新章节一致。
- 当前小说篇幅档位和章节字数范围已声明。
- 上一章 review 通过。
- 当前 Arc 的后续 3 章目标清晰。
- 当前 Arc 人物门禁已完成，关键人物画像和阶段命运方向已确认。
- 当前 Arc 关系线编排已完成，后续 3 章的关系线推进/维持/暂停节点清晰。
- 当前 Arc 事实显露节拍已完成，后续 3 章涉及的 Fact ID、显露层级和角色知情变化清晰。
- canon/facts.md 中活跃事实的揭露窗口、公开版本、知情状态和禁止改写项已补齐。
- 活跃伏笔没有逾期未处理项。
- 主角目标、敌人行动、分类定位和类型承诺未失焦。
