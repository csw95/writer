# Prose Style Methodology

叙事表现规范。用于 Planner、Writer 和 Reviewer，目标是提升场景画面感、动作颗粒度、视角沉浸和语言质感，而不是堆砌华丽词藻。

## 核心原则

1. **先呈现，后解释**: 重要信息优先通过人物动作、环境压力、物件状态、对话反应和结果变化显露。
2. **具体优先于抽象**: 少写“很危险、很愤怒、很混乱”，多写危险来自哪里、愤怒压在什么动作里、混乱如何改变选择。
3. **描写必须推动场景**: 每段描写至少服务氛围、冲突、人物、关系、伏笔、设定显露或行动判断中的一项。
4. **动作必须可视化**: 关键动作要让读者能看清起手、碰撞、受阻、变化和余波。
5. **视角必须稳定**: 描写应贴合当前 POV 的注意力、知识边界、情绪和身体感受，不越权展示角色不知道的信息。
6. **句式服务节奏**: 高压、战斗、追逐、争执使用更短更硬的句子；过渡、观察、压抑蓄势可适当放缓。
7. **正文语域纯净**: 正文区必须是读者可见文本，不得出现章节计划、状态台账、Fact/Loop/REL、Scene、STATE 或 AI 角色名等内部接力标记。

## Planner 呈现计划

每个关键场景必须在 chapter_plan 中补充呈现计划：

- **呈现焦点**: 本场景读者最该看见的画面、人物动作或物件状态。
- **感官锚点**: 至少 2 个具体感官线索，可来自声音、光线、气味、温度、触感、疼痛、压迫感或空间距离。
- **动作节拍**: 关键动作按“触发 -> 反应 -> 碰撞/受阻 -> 代价 -> 余波”拆解。
- **句式节奏**: 标明本场景主要句式策略，如短促、压迫、舒缓、克制、爆发、冷硬。
- **禁止概括**: 标明不能用一句话带过的关键动作、情绪或局势变化。

呈现计划不得引入未登记设定、未建档关键人物、未规划道具、未授权事实显露或未规划关系变化。

## Writer 执行标准

### 场景画面

- 每个关键场景开头应快速建立空间关系：谁在场、压力从哪里来、主角能看见或感受到什么。
- 环境描写应成为阻碍、线索、风险、情绪放大器或选择条件。
- 重要物件、伤势、资源和能力状态应通过可见变化呈现，而不是只用旁白声明。

### 动作颗粒度

关键动作应优先写成连续链条：

```text
触发 -> 身体反应 -> 主动动作 -> 阻碍/碰撞 -> 代价 -> 局势变化
```

动作场景避免只写结论：

- 不写：“他迅速击退对手。”
- 应写：先写对手如何逼近、主角如何避让或硬接、代价是什么、对手为何被迫后退、局势如何改变。

### 情绪外化

- 情绪优先落到动作、语气、停顿、视线、呼吸、握持、伤口、选择和反应上。
- 可写内心，但不能用大段内心独白替代行动与冲突。
- 角色压抑、犹豫、愤怒、恐惧或动摇时，必须让这种情绪影响他的判断或行动。

### 感官锚点

- 每个关键场景至少使用 2 个与冲突相关的感官锚点。
- 感官细节必须有叙事功能，不能变成装饰性堆砌。
- 同一章节中避免反复使用同一种意象、颜色、气味或形容方式。

### 句式节奏

- 高压动作：短句、断句、动词前置，减少解释。
- 信息辨认：中等句长，突出细节差异和角色判断。
- 情绪压抑：句子可更克制，留停顿和未说出口的反应。
- 爽点爆发：先收紧，再集中释放，不提前解释胜利原因。

### 叙述完整度

正文可以使用短句和断句，但关键叙述不能因为追求节奏而省掉动作完成点。读者必须能看清“谁 / 什么东西”发生了什么变化，以及这个变化是否已经完成。

- **动作句优先完整落点**: 物件或身体反应应写成 `主体 + 动作 + 完成点 / 余波`。例如 `布包里的断碑轻轻震了一下`、`墙上的灰光跟着震了一下`。避免把关键动作压成 `断碑轻轻震`、`灰光一震` 这类读感悬空的句子。
- **单字动词慎用收尾**: `震`、`响`、`冷`、`亮`、`沉`、`停` 等单字动词若用于关键线索，应补足 `了一下`、`响了一声`、`随之发冷`、`亮起一线`、`沉到掌心`、`停在纸边` 等完成信息。
- **省略句只用于强节奏点**: `不是封门。是下矿。` 这类判断式短句可以保留，因为它表达的是角色判断和节奏转折；但承载物件状态、能力反馈、伤势变化、证据显露的句子应优先写完整。
- **不要用说明文式补全**: 补完整不等于机械加“进行、产生、发生、震动”。优先使用故事内动作，如 `轻轻震了一下` 通常比 `发生轻微震动` 更自然。
- **修订门禁**: 顺读时若一句话需要读者自行补出谓语、宾语或动作结果，且它不是刻意的对话断句、心理停顿或判断重锤，应补成完整叙述。

### 词藻与人类表达

文字优化目标是让句子更像角色当下会感知、会判断、会说出的内容，而不是让旁白显得“会写”。正文修改优先遵守以下规则：

- **少用模板副词替代动作**: `猛地`、`微微`、`瞬间`、`顿时`、`下一刻`、`转眼`、`很快` 不能连续堆叠。需要速度或突然性时，优先写动作结果：`红线一缩`、`纸角立刻发白`、`手先一步按住匣角`。
- **少用模糊判断词**: `似乎`、`仿佛`、`好像`、`某种`、`某个` 只在 POV 确实无法确认、且这种不确定性服务悬疑或误判时使用。能具体写清的，应改成可见物、声音、气味、位置或角色判断。
- **避免机械心理词**: `下意识`、`本能地`、`不由得` 若只是填补动作动因，应改成身体反应或选择压力，例如“右手先一步按住匣角”“喉结动了一下”“手指扣进石缝”。
- **对话标记保持干净**: 少写 `淡淡道`、`冷冷道`、`平静地说道`。优先使用 `说`、`道`，再用停顿、视线、手势、动作或场面压力表现语气。
- **时间承接必须故事内化**: 正文不得写 `上一章`、`下一章`、`本章`、`Ch12` 等接力词。改为 `方才`、`先前`、`上一轮追索`、`接下来`、`若再耽搁` 等角色可感知的连续性表达。
- **抽象词必须落地**: `危险`、`强大`、`震撼`、`复杂`、`混乱`、`冰冷` 等可用，但不得单独承担描写功能。它们必须连接到具体来源、物件状态、身体感受或选择后果。
- **保留必要的口语和短句**: 易读优先于华丽。长句必须让因果顺序清楚；高压段落宁可拆短，也不要用成串修饰词遮住动作。

### 正文发布语域

- `## 正文` 到正文分隔线 `---` 之间只写故事内可见内容；不得写给 Planner、Reviewer 或 State Manager。
- 正文不得出现 `Ch19`、`Chapter 20`、`Scene 1`、`FACT-001`、`LOOP-001`、`REL-MAIN-001`、`STATE 2`、`chapter_plan`、`pending_action`、`run_lock`、`Writer AI`、`Reviewer AI`、`State Manager AI`、`novel_id` 等内部标记或系统术语。
- 当计划中出现章节编号或系统标记时，Writer 必须转译成故事内连续性表达：`Ch19` 可写成“方才”“刚才”“先前那道”“上一轮追索中”；`Ch21` 可写成“接下来”“下一步”“若再耽搁”；`FACT/LOOP/REL` 只转化为读者能看到的证据、伏笔、关系动作或人物反应。
- 元数据、场景执行记录、已显露事实、审阅报告和状态文件可以使用内部标记；正文区不可以。

## 禁止模式

- ❌ 用抽象形容词代替具体场面：危险、震撼、恐怖、强大、愤怒、混乱。
- ❌ 用总结句跳过关键过程：一番激战后、很快、转眼、众人震惊。
- ❌ 只写结果不写代价：获胜、逃脱、说服、突破没有过程和损耗。
- ❌ 环境描写与选择无关，只制造氛围。
- ❌ 感官细节堆叠但不改变人物判断或读者信息。
- ❌ POV 越权描写角色不知道的真相、动机或事实库内容。
- ❌ 同一章连续使用相同句式、相同反应和相同意象制造“流水账”。
- ❌ 用模板词制造 AI 腔：连续堆叠“猛地、微微、似乎、仿佛、某种、瞬间、顿时、下一刻、很快”等，而不写清具体动作和判断依据。
- ❌ 正文区泄漏内部标记：Ch/Chapter 编号、FACT/LOOP/REL ID、Scene/STATE、chapter_plan/state/canon 字段名、AI 角色名或运行控制字段。

## Reviewer 门禁

Reviewer 必须独立检查以下表现维度：

| 维度 | 说明 |
|------|------|
| scene_texture | 场景是否有可感知空间、压力、物件状态和氛围 |
| action_clarity | 关键动作是否清晰、连续、有碰撞、有代价 |
| prose_vividness | 语言是否具体、有辨识度，避免抽象总结 |
| pov_embodiment | 是否贴合 POV 的注意力、身体感受、知识边界和情绪状态 |
| body_hygiene | 正文区是否没有内部接力标记、工程标记或系统术语 |
| lexical_naturalness | 是否避免模板副词、模糊判断词、机械心理词和出戏时间词，语言是否自然易读 |

判定规则：

- 任一表现维度 < 5：对应场景必须重写。
- 任一表现维度为 5-6：正文必须小修后才能进入 State Manager。
- 全部表现维度 >= 7：叙事表现通过。
- `lexical_naturalness` 出现正文内部标记、明显 AI 腔高频重复或模糊词替代关键信息时，至少判为小修；若影响因果理解或角色判断，重写对应段落。
- 表现修正不得改变 chapter_plan 的事件结果、关系变化、事实显露、战力资源和时间地点连续性。
- `body_hygiene` 不打文学分；正文区只要出现内部标记即判为小修必改，若多处泄漏影响沉浸或语义成立则重写对应场景。
