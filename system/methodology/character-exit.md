# Character Exit Protocol

角色死亡、离队、长期缺席、背叛、洗白、退场和休眠都必须提前规划，不得临场处理。

## 触发范围

以下行为必须在 Volume / Arc / chapter_plan 中预设：

- 死亡
- 离队
- 长期缺席
- 背叛
- 洗白
- 阵营转向
- 关系确认或决裂
- 重要功能交接

## 退场前检查

- 该角色承担的主线功能是否已完成或转移。
- 该角色绑定的 Fact / LOOP / REL 是否已回收、转移或标记休眠。
- 该角色掌握的信息是否符合知情状态，不能带走未安排的关键证据。
- 退场是否改变主角目标、敌对势力计划或后续 3-5 章压力。
- 是否需要更新 cast-active 的休眠人物或移出活跃人物。

## 退场后更新

State Manager 必须同步：

- `characters/cast.md`
- `characters/cast-active.md`
- `state/current-state.md`
- `open-loops/loops.md`
- 相关 Fact 的知情状态
- 关系线状态

未完成这些同步时，不得进入下一章。
