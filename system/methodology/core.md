# Core Methodology

多小说 AI 长篇写作系统的公共方法论。此文件定义所有小说都必须遵守的基础写作约束。

## 总纲

长篇小说不是把一个故事写长，而是用“主线目标 + 阶段目标 + 人物成长 + 冲突升级 + 情绪兑现 + 伏笔回收”组成持续阅读系统。

## 十五个公共核心

1. **一句话卖点法**: 初始化时必须用一句话说明主角、极端处境、特殊优势/缺陷、目标、情绪回报。
2. **黄金三章法**: 前三章必须快速建立主角、处境、冲突、目标、期待感。
3. **三幕式结构**: 全书必须有开端、发展、结局，以及关键转折、中点、高潮。
4. **卷结构法**: 100 章以上小说必须拆成多个 Volume，每卷有阶段目标、敌人、奖励、代价。
5. **阶段目标法**: 终极目标拆成卷目标、Arc 目标、章节目标。
6. **人物欲望/需求法**: 核心人物必须有 Want、Need、Lie、Wound。
7. **冲突升级法**: 小麻烦 -> 大麻烦 -> 无法逃避 -> 必须选择 -> 付出代价 -> 新后果。
8. **场景五要素法**: 每个关键场景必须有目标、阻碍、危机选择、高潮行动、结果。
9. **期待-兑现法**: 每章建立期待，并至少兑现一个小期待或推进一个大期待。
10. **伏笔台账法**: 所有伏笔必须记录埋设、表面作用、真正作用、计划回收、最晚回收。
11. **地图/阶层升级法**: 长篇推进必须有空间、阶层、资源、敌人或认知层级的变化。
12. **分类定位法**: 初始化必须声明频道、主分类、细分类和标签，后续章节不得漂移。
13. **类型承诺法**: 每章、每 Arc、每卷都必须持续兑现题材核心快感。
14. **人物画像与命运法**: 关键人物登场前必须有最小画像；每个大情节前必须确定阶段人物画像和命运方向，后续只能渐进丰富，不能临场改写。
15. **关系线主次编排法**: 情感、手足、同袍、师徒、敌友和阵营信任等关系线必须提前确定主次、功能、节点和越界边界；凡影响主角选择、阵营站位、证据链、恋爱、背叛、牺牲、离队或救赎的关系推进，必须写入 long-term-arc、当前 Volume、当前 Arc 和 chapter_plan。

## 生命周期映射

| 生命周期 | 必须应用的方法论 | 产物 |
|----------|------------------|------|
| STATE 0 初始化 | 一句话卖点、分类定位、类型承诺、人物 Want/Need、人物命运纲要、世界规则 | premise、world、characters、state |
| STATE 1 结构生成 | 三幕式、卷结构、阶段目标、地图/敌人/能力升级、关系线主次编排 | long-term-arc、volume、arc |
| STATE 2 章节规划 | 场景五要素、冲突升级、期待-兑现、伏笔台账、章节关系线操作 | chapter_plan |
| STATE 3 正文生成 | 黄金三章、情绪曲线、场景转折、对话行动化 | chapter |
| STATE 4 审阅 | 类型承诺、节奏、连贯性、伏笔、钩子 | review |
| STATE 5 状态更新 | 状态连续性、伏笔回收、变量变化 | state、open_loops |

## 单章自动化前置门禁

任何单章自动化任务开始前，指定 `novel_id` 必须先通过 `system/templates/run-start-protocol.md`，并具备：

- `bible/premise.md` 已完整填写
- `bible/premise.md` 已声明 channel / category / subcategory / tags
- `world/worldbuilding.md` 已定义世界规则、资源、代价、禁止事项和开篇显露策略
- `characters/cast.md` 已定义主角 Want/Need/Lie/Wound、核心人物画像、敌对势力、关系网、阶段命运方向和新增人物规则
- `characters/cast.md` 或结构文件已定义关系线台账，包含情感、手足、同袍、师徒、敌友和阵营信任等长期关系线的主次、功能、阶段节点和越界边界
- `structure/long-term-arc.md` 已拆出全书阶段目标
- 至少一个 `structure/volumes/volume-XXX.md`
- 至少一个 `structure/arcs/arc-XXX.md`
- 当前 Volume 已定义本卷关键人物配置、关系变化和阶段命运方向
- 当前 Arc 已通过人物门禁，新增人物具备最小画像
- 当前 Volume / Arc 已定义本阶段关系线编排，且章节级关系节拍已明确
- `open-loops/loops.md` 具备伏笔台账格式
- 前三章目标已规划，或至少第一章 chapter_plan 已通过审阅
- `state/current-state.md` 已包含运行控制字段，且 `pending_action` 已正确路由
- `run_lock` 可用，且上次运行不是未恢复的 partial 状态

缺少以上任一项，只允许进行内容补齐、恢复或待办动作处理，不允许生成章节。
