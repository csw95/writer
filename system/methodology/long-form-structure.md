# Long Form Structure Methodology

长篇结构规范。用于支撑 short / medium / long / ultra_long 四档连续生成。

篇幅、章节数、单章字数和总字数必须先遵守 `system/methodology/length-standards.md`。

## 结构层级

```text
Book 终极目标
  -> Part 大阶段目标，仅 ultra_long 必填，通常 250-500 章
    -> Volume 卷目标，按 length_tier 通常 15-150 章
      -> Arc 阶段目标，按 length_tier 通常 5-50 章
      -> Chapter 章节目标，通常 1 章
        -> Scene 场景目标，通常 1-5 个关键场景
```

## 目标拆解

- **终极目标**: 读者长期期待，例如复仇、登顶、破案、建立势力、拯救世界。
- **Part 目标**: 超长篇中的大阶段目标，通常承载一次地图、阶层、规则或世界真相跃迁。
- **卷目标**: 当前大阶段必须完成的外部目标。
- **Arc 目标**: 按篇幅档位在 5-50 章内可兑现的剧情闭环。
- **章节目标**: 本章必须改变的状态。
- **场景目标**: 角色在当前场景想做的具体事。

## 长篇升级轴

每个 Volume 至少推进两个升级轴：

| 升级轴 | 示例 |
|--------|------|
| 地图升级 | 新手村 -> 宗门 -> 王朝 -> 大陆 -> 上界 |
| 阶层升级 | 底层 -> 小有名气 -> 核心圈 -> 影响世界 |
| 敌人升级 | 小反派 -> 组织 -> 宗门/集团 -> 幕后 Boss |
| 能力升级 | 技能、装备、资源、认知逐步提升 |
| 秘密升级 | 小谜团 -> 阶段真相 -> 世界真相 |
| 关系升级 | 陌生 -> 合作 -> 信任 -> 冲突 -> 重构 |

## 关系线结构轴

关系升级必须遵守 `system/methodology/relationship-lines.md`。任何会影响主角选择、阵营站位、证据链、牺牲、背叛、离队、救赎、恋爱推进或长期情绪回报的关系线，都要在结构层提前编排。

每本小说至少需要一张关系线主次表：

| 关系线ID | 类型 | 优先级 | 关联角色/阵营 | 叙事功能 | 全书阶段节点 | 越界禁止 |
|----------|------|--------|---------------|----------|--------------|----------|

超长篇还必须在 Part 层确认关系线是否推进、暂停、分裂、重构或回收。

## Volume 必填字段

- 卷名与章节范围
- 卷目标
- 分类服务点
- 类型承诺
- 本卷关键人物配置、关系变化和阶段命运方向
- 本卷关系线编排：本卷推进/维持/暂停的关系线、起点、终点、关键节点、越界禁止
- 地图/阶层变化
- 主要敌人和敌人主动行动
- 主角能力/资源变化
- 核心冲突阶梯
- 伏笔埋设与回收计划
- 卷高潮和卷尾钩子

## Part 必填字段（ultra_long）

- Part 名称与章节范围
- Part 级主线目标
- 地图/阶层/势力/规则跃迁
- 本 Part 的核心敌人或制度压力
- 关键 Volume 列表
- 主要人物在本 Part 的命运节点
- Part 级伏笔埋设与回收计划
- Part 终局事件和下一 Part 钩子

## Arc 必填字段

- Arc 名称与章节范围
- Arc 目标
- 分类服务点
- Arc 人物门禁：关键人物画像状态、叙事功能、关键选择和命运节点
- Arc 关系线编排：每条关系线的本 Arc 起点、终点、触发事件、章节级节拍和不推进章节
- Arc 内压力阶梯
- 每 3-5 章一个小爆点
- 本 Arc 至少回收 1 个伏笔或推进 1 个主线谜团
- Arc 结尾必须完成阶段目标，并引出更大问题

## 中期防散规则

连续 3 章不得只推进同一种爽点。

连续 5 章必须至少发生一个：

- 敌人主动升级
- 地图/场域变化
- 关系变化
- 资源获得或失去
- 伏笔揭示
- 主角认知变化

连续 10 章必须至少完成一个小闭环。

若连续 10 章没有任何关系线推进，下一次 Arc 规划必须明确这些关系线是“主动暂停”还是“结构遗漏”。若连续 3 章的关系推进没有服务主线目标、类型承诺或人物弧光，视为关系线抢戏风险，必须回到 Arc 规划调整。
