# Theme Methodology

主题追踪用于保证长篇不只是在事件升级，也在持续回答同一个核心问题。

## 主题声明

每本小说初始化时必须在 premise 或 long-term-arc 中写明：

- **核心主题问题**: 这本书反复拷问什么。
- **主角初始答案**: 主角一开始相信什么。
- **主角终局答案**: 主角最终要抵达什么理解。
- **反方答案**: 敌人或制度用什么错误答案压迫主角。
- **类型化表达**: 主题必须通过冲突、选择、代价、关系和事实显露呈现，不得写成说教。

## 章节级主题检查

chapter_plan 可在需要时增加：

```markdown
- **本章主题功能**: 推进/反证/误导/暂停/回收

- **反方答案施压**:
- **通过什么选择呈现**:
- **禁止说教项**:
```

## 主题漂移预警

出现任一情况，Reviewer 标记主题漂移风险：

- 连续 10 章没有任何选择触及核心主题。
- 爽点只剩机械胜利，不再改变主角对世界、关系或自我的理解。
- 反派压力和主题问题脱节。
- 关系线只制造情绪，不再服务主角答案变化。

主题漂移不直接阻塞单章，但必须在 stage_review_20ch 或下一次 Arc 规划中处理。
