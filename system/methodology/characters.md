# Character Methodology

人物画像与命运管理规范。适用于初始化、Volume/Arc 规划、章节规划、正文生成、审阅和状态更新。

## 目标

人物系统必须解决三个问题：

1. 读者能快速记住角色是谁。
2. 角色行动由欲望、恐惧、利益和关系驱动，而不是被剧情硬推。
3. 每个大情节开始前，关键人物的阶段作用和命运方向已经确定，后续可以逐步丰富但不能临场乱改。

## 人物分级

- **主角**: 必须完整画像，包含 Want / Need / Lie / Wound、选择模式、成长路线和阶段命运。
- **核心配角**: 长期反复登场，必须有独立目标、关系变化、秘密/伤痕和阶段命运。
- **长期反派/敌对代表**: 必须有目标、主动行动模式、压迫方式、弱点和阶段命运。
- **阶段人物**: 服务某个 Volume / Arc 的人物，登场前必须有最小画像和本阶段命运。
- **功能人物**: 只服务单一场景的信息、阻碍或氛围；若命名、反复登场或影响主线，必须升级为阶段人物并补齐画像。

## 登场前最小画像

任何会命名、反复登场、改变冲突结果、影响主角选择或承担伏笔功能的人物，登场前必须写入 `characters/cast.md`，至少包含：

- **角色ID/姓名**: 用于全文稳定引用。
- **人物级别**: 主角/核心配角/长期反派/阶段人物/功能人物。
- **身份/定位**: 一句话说明读者应如何记住他。
- **人设标签**: 2-5 个可感知标签。
- **叙事功能**: 推进、阻碍、信息提供、情感支撑、制造代价、制造误导等。
- **外在目标 Want**: 当前最想得到什么。
- **内在驱动/恐惧**: 为什么非要这样做。
- **与主角关系**: 初始关系和预期变化。
- **所属关系线**: 若角色参与长期情感、手足、同袍、师徒、敌友或阵营信任线，必须引用关系线ID。
- **资源/能力/限制**: 他能做什么，不能做什么。
- **说话与行动特征**: 至少 1 个可写作识别点。
- **首次登场计划**: 预计章节、入场场景、入场冲突。
- **阶段命运方向**: 本 Volume / Arc 内的转变、背叛、牺牲、死亡、离队、升阶、和解或伏笔保留。

核心配角、长期反派和主角还必须补充：

- **Need / Lie / Wound**: 内在需求、错误信念、伤痕来源。
- **关键秘密**: 如有，必须登记到 open-loops 或在 cast 中标记揭示时机。
- **不可违背底线**: 哪些行为会破坏人物一致性。
- **长期命运轨迹**: 入场状态、阶段转折、最终去向或暂定终局。

## 命运轨迹

命运不是必须提前写死每个细节，但必须提前确定“方向和边界”。

每个核心人物和阶段人物都必须有：

- **入场状态**: 登场时的处境、欲望、误解和压力。
- **关键选择**: 他在本阶段必须面对的选择。
- **转折节点**: 预计在哪个章节段发生立场、关系、能力或认知变化。
- **代价**: 选择后失去什么，或让谁付出代价。
- **阶段结局**: 本 Volume / Arc 结束时的状态。
- **最晚确认章节**: 若命运存在分支，必须在此章节前定稿。

允许保留 1-2 个命运分支，但分支必须有共同边界，不能在高潮前临时新增背叛、死亡、复活、洗白或强行恋爱。

## 关系线台账

`characters/cast.md` 必须为长期关系线建立台账，或明确引用结构文件中的同名台账。台账字段至少包含：

| 关系线ID | 类型 | 优先级 | 关联角色 | 当前状态 | 阶段目标 | 触发条件 | 下一节点 | 越界禁止 |
|----------|------|--------|----------|----------|----------|----------|----------|----------|

关系线ID建议格式：

- `REL-ROM-001`: 情感 / 恋爱线。
- `REL-KIN-001`: 亲情 / 手足线。
- `REL-COMRADE-001`: 同袍 / 盟友线。
- `REL-MENTOR-001`: 师徒 / 引路线。
- `REL-RIVAL-001`: 敌友 / 镜像线。
- `REL-FACTION-001`: 阵营 / 共同体线。
- `REL-MAIN-001`: 主角与制度 / 共同体 / 核心主题关系线。
- `REL-MIRROR-001`: 镜像人物 / 价值对照线。

以下关系变化必须提前进入关系线台账和当前 Volume / Arc：

- 恋爱升温、确认、误会、分离、复合。
- 手足和解、亲情决裂、救援、牺牲、传承。
- 同袍结盟、决裂、重盟、共同承担不可逆代价。
- 师徒传承、背叛、牺牲、断绝。
- 敌友合作、立场让步、阵营转向。
- 团队成立、内部分裂、组织规则成型。

## 大情节前人物门禁

每个 Part / Volume / Arc 开始前，必须先检查人物门禁：

- 本阶段会推动或阻碍主线的所有命名人物已经写入 `characters/cast.md`。
- 本阶段新增人物至少具备“登场前最小画像”。
- 本阶段核心人物的阶段命运方向已写明。
- 主角在本阶段的 Want / Need / Lie / Wound 推进点已明确。
- 关系变化、背叛、死亡、离队、救赎、升阶等重大命运节点已在 Volume / Arc 文件中登记。
- 情感、手足、同袍、师徒、敌友和阵营信任等关系线的主次、阶段节点、触发条件和越界禁止已在关系线台账及当前 Volume / Arc 中登记。
- 若某人物承担伏笔功能，必须同步登记到 `open-loops/loops.md`。

缺少以上任一项，只允许补齐人物画像和结构规划，不允许进入正文生成或自动化续写。

## 连载中新增人物

- 新增命名人物若只出现一次且不影响主线，可按功能人物建档处理，但不得承担关键转折。
- 新增人物若会再次出现、影响主角选择、改变冲突结果或承载伏笔，必须在章节规划前补入 `characters/cast.md`。
- 正文生成阶段不得临时新增关键人物；如确需新增，返回 STATE 2 前的内容补齐。
- 第二次登场前仍未建档的人物，不允许继续使用。
- 每章完成后，State Manager 必须把已登场人物的状态、关系变化、命运推进和已揭示秘密同步到 state / cast / open-loops。

## 渐进丰富规则

人物画像允许逐渐丰富，但人物纲要必须尽早确立：

- 登场前先锁定身份、目标、关系、功能和阶段命运。
- 登场后根据正文表现补充口癖、行为习惯、细节偏好和读者可感知特征。
- 重大转折前必须回查人物底线和命运轨迹，避免角色为剧情突然变脸。
- 画像补充只能解释和深化已发生行为，不得推翻既有选择逻辑。

## 禁止项

- 禁止未建档人物承担关键转折。
- 禁止没有动机的背叛、牺牲、死亡、洗白或恋爱推进。
- 禁止角色只作为设定说明工具存在。
- 禁止同类功能人物连续出现而无差异化人设。
- 禁止为了推进剧情临时改变人物 Want / Need / Lie / Wound。
- 禁止未登记关系线的恋爱升温、手足和解、同袍决裂、师徒传承、敌友让步或阵营转向承担关键剧情功能。
