# Key Items — {novel_id}

> 此文件在 STATE 0 初始化时填充。
> 参考: system/methodology/items.md、system/schemas/item.schema.md
> 只登记承担剧情功能的关键物品；普通消耗品只进入 state 资源变化。

## 使用规则

- 改变战斗、谈判、逃生、调查、证据链、人物选择或伏笔回收结果的物品，必须先登记为 `ITEM-*`。
- 物品真相、隐藏来源、底层规则必须同步登记到 `canon/facts.md` 的 `FACT-*`。
- 物品异常、误导、未来回收问题必须同步登记到 `open-loops/loops.md` 的 `LOOP-*` 或明确未来答案。
- State Manager 只能更新持有、位置、状态、生命周期和显露记录；不得临场改写真实功能、代价限制或禁止改写项。
