**结论**: 不通过完整开篇门禁，需先修订后再进入章节循环。

002 的高概念、类型承诺、人物画像、事实库和世界规则已经具备长篇支撑力。问题不在于设定缺失，而在于开篇时间轴与能力解锁规则存在硬冲突：第 3 章、第 30 章、第 100 章都在不同文件中承担“伪装死亡 / 逃出锈蚀三号 / 进入无主星海”的功能，导致苏翎、瓦伦、Volume 001 后半段无法自然接入。

**允许继续做的事**: 内容补齐、时间轴修订、黄金三章重排、伏笔 ID 补齐。

**暂不允许做的事**: 直接生成第一章后的连续章节、每日 3-10 章自动化、黄金三章批量写作。

说明：如果只写 Chapter 1 样章，当前设定能支撑“禁区回收 + 黑晶触发”的单章场景。但按本项目要求，写作前必须确认后续至少 3 章方向；当前 Ch2-Ch3 与 Arc/Volume 后续冲突，所以不建议启动正式章节循环。

## 通过项

### 1. 第一章开场条件足够强

Chapter 1 的基础动作链成立：

```text
底层回收任务 → 四十分钟氧气 → 禁区残骸 → 黑晶读数为零 → 陆沉主动触碰 → 残骸亮起
```

这符合开篇方法论中的“主角尽快出现、处境尽快明确、冲突尽快爆发、分类信号可感知”。第一章只要严格从陆沉当下处境写，不用百科式解释理事会和收割循环，就能成立。

### 2. 长篇核心矛盾清楚

主线不是单纯升级，而是：

```text
限制文明压迫 → 坟场遗言获取 → 源质真相揭露 → 势力建设 → 收割循环对抗 → 银河秩序重写
```

这条轴线能支撑 ultra_long 体量。

### 3. 人物和关系线基本达标

陆沉的 Want / Need / Lie / Wound 清楚；老莫牺牲、零的引路、苏翎的同袍线、瓦伦的敌友线、理事会制度对抗线都有明确的阶段位置。当前不需要新增关键人物才能启动开篇修订。

### 4. 事实库强于普通初始化

事实库已经把源质本质、收割循环、坟场先驱、人类基因锁、灰烬文明、老莫芯片、苏翎清除任务、卡伦恐惧、瓦伦母星等都建立为稳定 FACT。后续只要 chapter_plan 严格引用 Fact ID，长篇不会轻易失控。

## 阻塞问题

### P0-1: 逃出锈蚀三号的时间轴冲突

当前存在三套互相冲突的安排：

1. `premise` 黄金三章要求 Chapter 3 “伪装死亡逃出回收站”并截获收割预备清单。
2. `arc-001` 要求 Chapter 29-30 “最终逃亡、离开锈蚀三号”，下一 Arc 已在前往无主星海的货船上。
3. `volume-001` 又要求苏翎 Chapter 45-100 在锈蚀三号医疗线登场，瓦伦 Chapter 60-100 抵达锈蚀三号，卷末 Chapter 80-100 才完成逃亡。

**影响**:

- 如果 Ch30 已离开锈蚀三号，苏翎 Ch45-50 在锈蚀三号医疗站救陆沉就无法发生。
- 瓦伦 Ch60-80 抵达锈蚀三号时，主角已经离站，追捕线只能变成事后调查，无法提供本卷追捕压力。
- Volume 001 的 100 章结构被掏空，Arc 001 抢走了卷高潮。

**建议修订**:

保留 Volume 001 为 Ch1-100。将 Arc 001 的结尾从“逃出锈蚀三号”改为“逃出禁区残骸 / 躲入锈蚀三号下层 / 全站封锁”。最终“伪装死亡并进入无主星海”保留到 Ch80-100 卷高潮。

### P0-2: 灰烬遗言三层解锁规则冲突

世界规则规定：

```text
第一层：认知遗产
第二层：基因遗产，例如灰烬视觉
第三层：技术遗产，例如可复制设备、材料、算法或能源方案
```

但当前黄金三章与 Arc 001 写法中：

- Ch2 说“获得第一层基因解锁：灰烬视觉”。
- Ch3 让陆沉使用“改写源质标记的装置”伪装死亡。
- Ch26-28 又说“灰烬遗言第二层解锁，获得源质标记改写装置”。
- Arc 收束又说“遗言解锁到第二层（认知+基因），但已掌握源质标记改写初阶使用”。

**影响**:

- 灰烬视觉到底是第一层、第二层，还是临时后遗症不清楚。
- 标记改写装置属于技术遗产，却提前在 Ch3 或 Ch29-30 使用，破坏“第三层技术遗产”的资源门槛。
- 源质烧伤作为代价的触发时机不稳定，后续状态难以追踪。

**建议修订**:

统一为以下节奏：

| 阶段 | 建议章节 | 内容 | 规则状态 |
|------|----------|------|----------|
| 黑晶触发 | Ch1 | 坟场共鸣启动，残骸亮起 | 未正式解锁 |
| 认知遗产 | Ch2 | 灰烬文明灭亡记忆冲击 | 第一层 |
| 不稳定后遗症 | Ch2-Ch5 | 左眼短暂看见源质流和收割标记 | 不算正式灰烬视觉 |
| 稳定灰烬视觉 | Ch20-Ch30 或 Ch45 前 | 灰烬视觉稳定，作为基因遗产 | 第二层 |
| 标记改写技术 | Ch80-Ch100 | 卷高潮伪装死亡、逃入无主星海 | 第三层或一次性残片技术 |

若必须在 Ch3 给爽点，可把 Ch3 的“改写源质标记”降级为“黑晶残响造成的一次性误判”，不应写成可复制、可掌握的灰烬技术。

### P0-3: Ch3 信息揭露过早

当前 Ch3 章末钩子直接写：

```text
人类，收割优先度：最高——因为人类是三千年前上一轮收割中唯一逃脱的文明后裔
```

这与以下设定冲突：

- 世界显露策略要求暂不说明人类祖先文明的自我退化历史。
- 伏笔台账计划“人类祖先自我退化基因锁”在 Ch100 才揭示核心信息。
- FACT-011 计划 Ch80-100 才由零揭示“第一个进入坟场的人类祖先”。

**影响**:

Ch3 如果让读者直接知道“上一循环逃脱者后裔”，Ch80-100 的身份反转会失去力量。陆沉也会过早从“底层逃亡者”跃迁为“全人类真相承担者”，压缩成长弧线。

**建议修订**:

Ch3 只保留破碎信息，不给完整解释。例如：

```text
人类 / 优先度：最高 / 上一循环 / 逃脱样本 / 待清除
```

陆沉只能理解“人类在某张名单首位”，无法理解“上一循环逃脱者后裔”的全部含义。完整解释仍交给 Ch80-100 的零与事实库推进。

## 需修问题

### P1-1: 零的首次登场位置需要锁定

多处设定将零的首次登场安排在 Ch5，但黄金三章的灰烬文明遗言体验发生在 Ch2。可以保留这个设计，但要明确：

- Ch2 只写灰烬文明灭亡记忆和不具名“过滤感”，零不直接对话。
- Ch5 才让零首次以可辨认“声音”出现。

如果希望零在 Ch2 就救陆沉，则需要同步更新 `characters/cast.md`、`canon/facts.md`、`arc-001.md` 的首次登场章节。

### P1-2: open-loops 缺少稳定 LOOP ID

伏笔台账目前用文字名记录伏笔，canon 的 `related_loops` 也引用文字名。为了后续章节计划可追踪，建议改为：

```text
LOOP-001: 收割标记
关联 Fact: FACT-006, FACT-005
埋设: Ch2
计划推进: Ch50-80
最晚回收: Ch150
```

所有长期伏笔都应有 `LOOP-*`，并绑定至少一个 `FACT-*` 或明确未来答案。

### P1-3: current-state 的冲突解决章需同步

当前状态写“陆沉在回收站的底层压迫 expected_resolution_chapter: 30”。如果采用推荐方案，把最终逃出锈蚀三号放回 Ch80-100，则此项应改为：

- Ch30: 底层压迫第一次破局，进入全站封锁和逃亡阶段。
- Ch80-100: 锈蚀三号压迫线正式闭环，进入无主星海。

### P1-4: 黄金三章承载过重

目前前三章同时承担：

```text
底层压迫 + 黑晶觉醒 + 灰烬文明灭亡 + 灰烬视觉 + 收割标记 + 改写标记装置 + 伪装死亡 + 收割预备清单 + 人类逃脱者后裔
```

这会让开篇读者获得刺激，但也会挤压长篇悬念。建议前三章只完成：

```text
Ch1: 底层处境 + 黑晶异常
Ch2: 灰烬灭亡记忆 + 不稳定灰烬视觉 + 收割标记
Ch3: 回收队围捕 + 一次性脱身 + 破碎清单片段
```

正式技术遗产、身份反转和最终离站后移。

## 推荐修订顺序

1. **先锁定 Volume 001 时间轴**: Ch1-30 留在锈蚀三号内部逃亡，Ch45-50 苏翎医疗线，Ch60-80 瓦伦追捕线，Ch80-100 最终伪装死亡离站。
2. **重写黄金三章表**: 将 Ch3 的“逃出回收站”改为“逃出残骸/进入下层/全站封锁”，将“逃脱者后裔”改为破碎信息。
3. **建立灰烬遗言解锁表**: 明确每一层对应章节、能力、资源、代价和显露层级。
4. **同步 Arc 001 收束**: 将 Arc 001 的结尾改为“锈蚀三号内部阶段胜利”，不要让它抢走 Volume 001 卷高潮。
5. **补 LOOP ID 与 Fact 绑定**: 让伏笔、事实库、章节计划之间可稳定引用。
6. **更新 current-state**: 修正当前 Arc/Volume 的阶段目标和冲突预计闭环章节。
